BÀI 1 — LỚP (CLASS) & KẾ THỪA TRONG PYTHON

1.1. LỚP (CLASS) LÀ GÌ?

Lớp (Class) là "bản thiết kế" để tạo ra các đối tượng (object). Lớp quy định: đối tượng sẽ có thuộc tính gì (dữ liệu) và phương thức gì (hành động).

💡 Ví dụ đời thường: Lớp = "Bản vẽ kỹ thuật chiếc xe". Đối tượng = "Chiếc xe Toyota mang biển 51A-12345" được dựng theo bản vẽ đó. Thuộc tính = màu, biển số. Phương thức = chạy, phanh, bóp còi.

1.2. Cú pháp khai báo lớp trong Python

class TenLop: def __init__(self, tham_so_1, tham_so_2): # Hàm tạo self.thuoc_tinh_1 = tham_so_1 # Gán thuộc tính self.thuoc_tinh_2 = tham_so_2 def phuong_thuc(self): # Phương thức print(f"Thuộc tính 1 = {self.thuoc_tinh_1}") # Sử dụng: obj = TenLop("giá trị 1", 123) # Tạo đối tượng obj.phuong_thuc() # Gọi phương thức

1.4. Lớp MatHang trong đồ án

models/item.py class MatHang: def __init__(self, ma_so, ten, gia_co_ban, so_luong): # Ép kiểu an toàn để tránh lỗi nhập liệu self.ma_so = str(ma_so).strip() self.ten = str(ten).strip() self.gia_co_ban = float(gia_co_ban) self.so_luong = int(so_luong) def tinh_gia_ban(self): # Mặc định: giá bán = giá cơ bản (lớp con sẽ GHI ĐÈ) return self.gia_co_ban def to_dict(self): return { "loai": "MatHang", "ma_so": self.ma_so, "ten": self.ten, "gia_co_ban": self.gia_co_ban, "so_luong": self.so_luong }

Giải thích từng dòng:

  • class MatHang: — khai báo lớp MatHang.
  • def __init__(self, ma_so, ten, gia_co_ban, so_luong): — hàm tạo có 4 tham số.
  • self.ma_so = str(ma_so).strip() — ép ma_so thành chuỗi, rồi .strip() cắt khoảng trắng đầu/cuối. Ép kiểu an toàn để tránh lỗi nếu user nhập số.
  • self.gia_co_ban = float(gia_co_ban) — ép thành số thực (vì giá có thể có lẻ).
  • self.so_luong = int(so_luong) — ép thành số nguyên.
  • def tinh_gia_ban(self): — phương thức tính giá bán. Mặc định trả về gia_co_ban. Lớp con sẽ ghi đè.
  • def to_dict(self): — chuyển đối tượng thành dictionary (dict) để dễ lưu vào SQLite.

1.5. KẾ THỪA (Inheritance) LÀ GÌ?

Kế thừa là việc 1 lớp con (subclass) thừa hưởng toàn bộ thuộc tính và phương thức của lớp cha (superclass). Lớp con có thể bổ sung thuộc tính/phương thức mới, hoặc ghi đè (override) phương thức cũ.

# Cú pháp: class LopCon(LopCha): class Sach(MatHang): # Sach kế thừa MatHang def __init__(self, ma_so, ten, gia_co_ban, so_luong, tac_gia, nha_xuat_ban): super().__init__(ma_so, ten, gia_co_ban, so_luong) # Gọi __init__ lớp cha self.tac_gia = tac_gia # Thuộc tính riêng của Sach self.nha_xuat_ban = nha_xuat_ban

Giải thích:

  • class Sach(MatHang): — khai báo Sach kế thừa từ MatHang.
  • super().__init__(...) — gọi hàm tạo của lớp cha (MatHang). Bắt buộc để khởi tạo các thuộc tính chung (ma_so, ten, gia_co_ban, so_luong).
  • Sau đó thêm 2 thuộc tính riêng: tac_gia, nha_xuat_ban — chỉ Sách mới có.

BÀI 2 — TÍNH ĐA HÌNH (POLYMORPHISM)

⚠️ PHẦN NÀY HAY BỊ HỎI THI NHẤT! Đọc kỹ, học thuộc 5 công thức tính giá.

2.1. Đa hình LÀ GÌ?

Đa hình (Polymorphism) — cùng 1 lời gọi phương thức nhưng mỗi đối tượng trả về kết quả khác nhau tùy loại. Đa hình đạt được bằng cách các lớp con ghi đè (override) phương thức của lớp cha.

💡 Ví dụ đời thường: Cùng câu lệnh "Hãy kêu!" nhưng: Chó kêu "Gâu gâu", Mèo kêu "Meo meo", Vịt kêu "Quạc quạc". Cùng 1 lời gọi kêu() nhưng mỗi loài trả về âm thanh khác nhau → ĐA HÌNH.

2.2. GHI ĐÈ (Override) LÀ GÌ?

Ghi đè = lớp con định nghĩa lại phương thức đã có ở lớp cha, với logic riêng của lớp con. Khi gọi, Python sẽ dùng phiên bản của lớp con (không phải lớp cha).

2.3. 5 công thức tinh_gia_ban() trong đồ án

2.4. Code thực tế

models/subclasses.py class Sach(MatHang): def tinh_gia_ban(self): return self.gia_co_ban * 1.20 # +20% class TapChi(MatHang): def tinh_gia_ban(self): return self.gia_co_ban * 0.90 # -10% class Bao(MatHang): def tinh_gia_ban(self): return self.gia_co_ban * 1.00 # giữ nguyên class LuanVan(MatHang): def tinh_gia_ban(self): return self.gia_co_ban * 1.30 # +30% class BanThao(MatHang): def tinh_gia_ban(self): if self.tinh_trang.lower() == "mới": return self.gia_co_ban * 1.50 # +50% (mới) else: return self.gia_co_ban * 1.20 # +20% (cũ)

2.5. Cách đa hình hoạt động

# Tạo 5 đối tượng khác loại ds = [ Sach("S01", "Sách A", 100000, 10, "TG", "NXB"), TapChi("T01", "Tạp chí B", 100000, 5, "Số 1"), Bao("B01", "Báo C", 100000, 20, "01/01/2025"), LuanVan("LV01", "Luận văn D", 100000, 2, "ĐH Bách Khoa"), BanThao("BT01", "Bản thảo E", 100000, 1, "mới"), ] # CÙNG 1 lời gọi tinh_gia_ban() — 5 kết quả khác nhau! for sp in ds: print(sp.tinh_gia_ban()) # In ra: 120000, 90000, 100000, 130000, 150000
💡 Đây chính là ĐA HÌNH! Cùng sp.tinh_gia_ban() nhưng mỗi sp trả về giá khác nhau tùy loại. Python tự biết gọi đúng phương thức của lớp con tương ứng.

BÀI 3 — HÀNG ĐỢI (QUEUE) — FIFO

3.1. Queue LÀ GÌ?

Hàng đợi (Queue) là cấu trúc dữ liệu theo nguyên tắc FIFO — First In, First Out (Vào trước, ra trước).

💡 Tưởng tượng đời thường: Hãy nghĩ về xếp hàng mua vé. Người đến trước được phục vụ trước. Người đến sau phải xếp cuối hàng, đợi đến lượt.

3.2. So sánh Stack và Queue

Đặc điểmStack (LIFO)Queue (FIFO)
Nguyên tắcVào sau, ra trướcVào trước, ra trước
Ví dụChồng đĩaXếp hàng mua vé
Số đầu1 (đỉnh)2 (đầu + cuối)
Ứng dụngUndo/RedoGiỏ hàng + thanh toán

3.3. 3 thao tác chính

Tên tiếng AnhTên codeTác dụng
ENQUEUEthem_vao(du_lieu)Thêm phần tử vào CUỐI hàng
DEQUEUElay_ra()Lấy phần tử ở ĐẦU hàng ra
PEEK(không có riêng)Xem phần tử đầu không lấy

3.4. Cài đặt trong đồ án (file models/hang_doi.py)

3.4.1. Lớp NutHangDoi

class NutHangDoi: def __init__(self, du_lieu): self.du_lieu = du_lieu self.sau = None # Trỏ tới mắt xích đứng SAU mình

3.4.2. Lớp HangDoi

class HangDoi: def __init__(self): self.dau = None # Phần tử ĐẦU (lấy ra ở đây) self.cuoi = None # Phần tử CUỐI (thêm vào ở đây) self.so_luong = 0

Khác Stack: Queue cần 2 con trỏ dau và cuoi vì thao tác ở 2 đầu khác nhau.

3.4.3. Hàm them_vao() — ENQUEUE

def them_vao(self, du_lieu): nut_moi = NutHangDoi(du_lieu) if self.cuoi is None: # Hàng rỗng → phần tử mới là vừa đầu vừa cuối self.dau = nut_moi self.cuoi = nut_moi else: self.cuoi.sau = nut_moi # Phần tử cuối hiện tại trỏ tới phần tử mới self.cuoi = nut_moi # Cập nhật lại con trỏ `cuoi` self.so_luong += 1

3.4.4. Hàm lay_ra() — DEQUEUE

def lay_ra(self): if self.dau is None: return None # Hàng rỗng nut_lay = self.dau # Lưu phần tử đầu self.dau = self.dau.sau # Đầu mới = phần tử đứng sau if self.dau is None: self.cuoi = None # Nếu lấy hết → cuoi cũng None self.so_luong -= 1 return nut_lay.du_lieu

3.5. Ứng dụng: GIỎ HÀNG + THANH TOÁN theo FIFO

Trong đồ án, giỏ hàng là 1 Queue. Khi user bấm "Thêm vào giỏ" → gọi them_vao(sp). Khi bấm "Thanh Toán" → vòng lặp while not rong(): lấy từng món ra theo FIFO, món nào thêm trước sẽ thanh toán trước.

# Hàm thanh toán (trong hop_thoai_gio_hang.py) while not self.cua_so_cha.gio_hang.rong(): sp = self.cua_so_cha.gio_hang.lay_ra() # Lấy món ĐẦU tiên sp_trong_kho = self.cua_so_cha.danh_sach.tim_kiem_theo_ma(sp.ma_so) if sp_trong_kho is not None and sp_trong_kho.so_luong > 0: sp_trong_kho.so_luong -= 1 # Giảm tồn kho tong_tien += sp.tinh_gia_ban() # Đa hình!

BÀI 4 — PYQT6 DIALOGS (Form Thêm/Sửa, Giỏ hàng, Đăng nhập)

4.1. QDialog LÀ GÌ?

QDialog là cửa sổ pop-up phụ, thường dùng để thu thập thông tin từ người dùng. Khác với QMainWindow (cửa sổ chính), dialog có 2 kết quả khi đóng: Accepted (bấm Lưu) hoặc Rejected (bấm Hủy).

4.2. Cấu trúc cơ bản 1 QDialog

from PyQt6.QtWidgets import QDialog, QVBoxLayout, QPushButton class HopThoaiCuaToi(QDialog): def __init__(self, parent=None): super().__init__(parent) self.setWindowTitle("Hộp thoại của tôi") self.resize(400, 300) self.thiet_lap_giao_dien() def thiet_lap_giao_dien(self): layout = QVBoxLayout() layout.addWidget(QLabel("Nhập gì đó:")) self.nhap = QLineEdit() nut_luu = QPushButton("Lưu") nut_luu.clicked.connect(self.chap_nhan) # → Accepted nut_huy = QPushButton("Hủy") nut_huy.clicked.connect(self.reject) # → Rejected layout.addWidget(self.nhap) layout.addWidget(nut_luu) layout.addWidget(nut_huy) self.setLayout(layout) def chap_nhan(self): # Kiểm tra dữ liệu rồi mới accept if not self.nhap.text().strip(): QMessageBox.warning(self, "Lỗi", "Không được bỏ trống!") return self.accept() # Đóng dialog với kết quả Accepted # Cách gọi từ cửa sổ chính: hop_thoai = HopThoaiCuaToi(self) if hop_thoai.exec() == QDialog.DialogCode.Accepted: # User bấm Lưu text = hop_thoai.nhap.text() print(f"Đã nhập: {text}")

4.3. HopThoaiThemMoi — form Thêm/Sửa sản phẩm

Đặc điểm:

  • Combo chọn loại (5 loại: Sách/Tạp chí/Báo/Luận văn/Bản thảo)
  • 4 ô chung: mã, tên, giá, số lượng
  • 2 ô riêng (ẩn/hiện theo loại — vd: Sách có Tác giả + NXB, Tạp chí chỉ có Số phát hành)
  • Kiểm tra dữ liệu: bắt lỗi bỏ trống, sai định dạng, sai tình trạng
  • Hàm thay_doi_loai() — tự ẩn/hiện ô nhập riêng khi user đổi loại

4.4. HopThoaiGioHang — form Giỏ hàng

Đặc điểm:

  • Bảng QTableWidget hiển thị giỏ (theo thứ tự FIFO)
