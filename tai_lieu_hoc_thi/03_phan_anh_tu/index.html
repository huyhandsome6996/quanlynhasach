BÀI 1 — LỚP (CLASS) & KẾ THỪA TRONG PYTHON NGỌ phụ trách

📌 Lưu ý: OOP do Ngọ phụ trách trong code. Nhưng Tú vẫn cần hiểu vì thi có thể hỏi chéo — đọc lướt cho hiểu sơ.

1.1. LỚP (CLASS) LÀ GÌ?

Lớp (Class) là "bản thiết kế" để tạo ra các đối tượng (object). Lớp quy định: đối tượng sẽ có thuộc tính gì (dữ liệu) và phương thức gì (hành động).

💡 Ví dụ đời thường: Lớp = "Bản vẽ kỹ thuật chiếc xe". Đối tượng = "Chiếc xe Toyota mang biển 51A-12345" được dựng theo bản vẽ đó. Thuộc tính = màu, biển số. Phương thức = chạy, phanh, bóp còi.

🚗 Ví dụ đời thường 1: Bản vẽ xe & chiếc xe cụ thể

Hãy tưởng tượng bạn làm kỹ sư tại nhà máy Toyota. Bạn có 1 bản vẽ kỹ thuật (Class) của dòng xe Camry. Bản vẽ quy định: xe có màu, biển số, động cơ; có thể chạy, phanh, bóp còi. Khi khách đặt xe, nhà máy dựng từng chiếc xe cụ thể (Object) theo bản vẽ đó — mỗi chiếc có màu và biển số riêng, nhưng đều có cùng cấu trúc.

1.4. Lớp MatHang trong đồ án (file lop_doi_tuong.py)

ung_dung_giao_tiep/loi_thuat_toan/lop_doi_tuong.py
from abc import ABC, abstractmethod

class MatHang(ABC):
    """Lớp cha trừu tượng đại diện cho 1 mặt hàng."""

    def __init__(self, ma_so, ten_san_pham, gia_co_ban, ton_kho):
        self.ma_so        = ma_so
        self.loai_hang    = ''
        self.ten_san_pham = ten_san_pham
        if gia_co_ban < 0:
            raise ValueError('Giá cơ bản không được âm.')
        self.gia_co_ban = gia_co_ban
        if ton_kho < 0:
            raise ValueError('Tồn kho không được âm.')
        self.ton_kho = ton_kho

    @abstractmethod
    def tinh_gia_ban(self):
        """Phương thức trừu tượng — lớp con PHẢI ghi đè."""
        pass

    def chuyen_thanh_dict(self):
        """Chuyển thành dict để trả về JSON."""
        return {
            'ma_so':        self.ma_so,
            'loai_hang':    self.loai_hang,
            'ten_san_pham': self.ten_san_pham,
            'gia_co_ban':   self.gia_co_ban,
            'ton_kho':      self.ton_kho,
            'gia_ban':      self.tinh_gia_ban()   # Đa hình!
        }

Giải thích từng dòng:

  • from abc import ABC, abstractmethod — nhập 2 công cụ OOP trừu tượng. ABC = Abstract Base Class.
  • class MatHang(ABC): — kế thừa từ ABC → MatHang là lớp trừu tượng, không thể tạo đối tượng trực tiếp.
  • if gia_co_ban < 0: raise ValueError(...) — kiểm tra đầu vào. Nếu giá âm → ném lỗi. Đây là đóng gói (encapsulation) — bảo vệ dữ liệu hợp lệ.
  • @abstractmethod — decorator đánh dấu phương thức trừu tượng. Lớp con bắt buộc phải ghi đè, nếu không sẽ lỗi khi tạo đối tượng.
  • def tinh_gia_ban(self): pass — body rỗng. Lớp con sẽ định nghĩa lại.
  • def chuyen_thanh_dict(self): — chuyển đối tượng thành dict (cho JSON). Gọi self.tinh_gia_ban() — đây là đa hình: Python tự gọi đúng hàm của lớp con.

1.5. KẾ THỪA (Inheritance) LÀ GÌ?

Kế thừa là việc 1 lớp con (subclass) thừa hưởng toàn bộ thuộc tính và phương thức của lớp cha (superclass). Lớp con có thể bổ sung thuộc tính/phương thức mới, hoặc ghi đè (override) phương thức cũ. 

# Cú pháp: class LopCon(LopCha):
class Sach(MatHang):              # Sach kế thừa MatHang
    def __init__(self, ma_so, ten_san_pham, gia_co_ban, ton_kho, tac_gia, nha_xuat_ban):
        super().__init__(ma_so, ten_san_pham, gia_co_ban, ton_kho)  # Gọi __init__ lớp cha
        self.loai_hang    = 'Sách'
        self.tac_gia      = tac_gia                # Thuộc tính riêng của Sach
        self.nha_xuat_ban = nha_xuat_ban

Giải thích:

  • class Sach(MatHang): — khai báo Sach kế thừa từ MatHang.
  • super().__init__(...) — gọi hàm tạo của lớp cha (MatHang). Bắt buộc để khởi tạo các thuộc tính chung.
  • Sau đó thêm 2 thuộc tính riêng: tac_gia, nha_xuat_ban — chỉ Sách mới có.

BÀI 2 — TÍNH ĐA HÌNH (POLYMORPHISM) NGỌ phụ trách

⚠️ PHẦN NÀY HAY BỊ HỎI THI NHẤT! Đọc kỹ, học thuộc 5 công thức tính giá.

2.1. Đa hình LÀ GÌ?

Đa hình (Polymorphism) — cùng 1 lời gọi phương thức nhưng mỗi đối tượng trả về kết quả khác nhau tùy loại. Đa hình đạt được bằng cách các lớp con ghi đè (override) phương thức của lớp cha.

💡 Ví dụ đời thường: Cùng câu lệnh "Hãy kêu!" nhưng: Chó kêu "Gâu gâu", Mèo kêu "Meo meo", Vịt kêu "Quạc quạc". Cùng 1 lời gọi kêu() nhưng mỗi loài trả về âm thanh khác nhau → ĐA HÌNH.

2.2. GHI ĐÈ (Override) LÀ GÌ?

Ghi đè = lớp con định nghĩa lại phương thức đã có ở lớp cha, với logic riêng của lớp con. Khi gọi, Python sẽ dùng phiên bản của lớp con (không phải lớp cha).

BÀI 3 — HÀNG ĐỢI (QUEUE) — FIFO TÚ

3.1. Queue LÀ GÌ?

Hàng đợi (Queue) là cấu trúc dữ liệu theo nguyên tắc FIFO — First In, First Out (Vào trước, ra trước).

💡 Tưởng tượng đời thường: Hãy nghĩ về xếp hàng mua vé. Người đến trước được phục vụ trước. Người đến sau phải xếp cuối hàng, đợi đến lượt.

3.2. So sánh Stack và Queue

3.4.2. Lớp HangDoi

class HangDoi:
    def __init__(self):
        self.dau       = None     # Phần tử ĐẦU (lấy ra ở đây)
        self.cuoi_hang = None     # Phần tử CUỐI (thêm vào ở đây)
        self.so_luong  = 0

Khác Stack: Queue cần 2 con trỏ dau và cuoi_hang vì thao tác ở 2 đầu khác nhau (thêm cuối, lấy đầu).

3.4.3. Hàm them_vao() — ENQUEUE

def them_vao(self, data):
    node_moi = NodeHangDoi(data)

    # Trường hợp 1: Hàng đợi rỗng
    if self.dau is None:
        self.dau       = node_moi
        self.cuoi_hang = node_moi
    else:
        # Trường hợp 2: Hàng đợi đã có phần tử
        self.cuoi_hang.sau = node_moi     # Node cuối hiện tại trỏ tới Node mới
        self.cuoi_hang    = node_moi      # Cập nhật lại con trỏ `cuoi_hang`

    self.so_luong += 1

Giải thích từng dòng:

  • node_moi = NodeHangDoi(data) — tạo Node mới chứa dữ liệu.
  • if self.dau is None: — kiểm tra hàng rỗng.
  • Nếu rỗng: self.dau = node_moi và self.cuoi_hang = node_moi — Node mới là vừa đầu vừa cuối.
  • Nếu không rỗng: self.cuoi_hang.sau = node_moi — Node cuối hiện tại trỏ sau tới Node mới.
  • self.cuoi_hang = node_moi — cập nhật cuoi_hang = Node mới.
  • self.so_luong += 1 — tăng bộ đếm.

3.4.4. Hàm lay_ra() — DEQUEUE

def lay_ra(self):
    if self.dau is None:
        return None                # Hàng rỗng

    data = self.dau.data           # Lưu dữ liệu đầu
    self.dau = self.dau.sau        # Đầu mới = phần tử đứng sau

    # Nếu hàng đợi chỉ còn 1 phần tử, cập nhật lại cuối
    if self.dau is None:
        self.cuoi_hang = None

    self.so_luong -= 1
    return data

Giải thích từng dòng:

  • if self.dau is None: — kiểm tra hàng rỗng.
  • data = self.dau.data — lưu dữ liệu của Node đầu vào biến tạm. Phải làm trước khi đổi self.dau.
  • self.dau = self.dau.sau — dịch dau sang Node tiếp theo. Node đầu cũ bị "bỏ rơi".
  • if self.dau is None: self.cuoi_hang = None — nếu sau khi lấy ra hàng rỗng, phải reset cuoi_hang = None để tránh tham chiếu treo.
  • return data — trả dữ liệu đã lưu.

3.5. Ứng dụng: GIỎ HÀNG + THANH TOÁN theo FIFO

Trong đồ án, giỏ hàng là 1 Queue. Khi user bấm "Thêm vào giỏ" → gọi them_vao(sp). Khi bấm "Thanh Toán" → vòng lặp while not rong(): lấy từng món ra theo FIFO, món nào thêm trước sẽ thanh toán trước. 

ung_dung_giao_tiep/loi_thuat_toan/undo_redo_gio_hang.py
class GioHang:
    """Lớp Giỏ hàng sử dụng Queue (FIFO)."""

    def thanh_toan(self):
        """Dequeue tất cả - Xử lý thanh toán và trả về hóa đơn."""
        if self.la_rong():
            return None
        hoa_don = {'cac_mon': [], 'tong_tien': 0, 'so_mon': self._so_luong}
        while not self.la_rong():
            mon = self._dau.du_lieu                # Lấy dữ liệu Node đầu
            hoa_don['cac_mon'].append(mon)         # Thêm vào hóa đơn
            hoa_don['tong_tien'] += mon['thanh_tien']
            self._dau = self._dau.tiep_theo        # Dịch đầu sang Node sau
            self._so_luong -= 1
        self._cuoi = None                          # Reset cuối
        return hoa_don

3.6. Mô phỏng Queue — giỏ hàng + thanh toán FIFO

🎟️ Ví dụ đời thường: Xếp hàng mua vé xem phim

Hãy tưởng tượng bạn đứng trước rạp chiếu phim. Có 1 hàng người xếp hàng mua vé (Queue). Cô nhân viên thu ngân sẽ phục vụ người đứng ĐẦU hàng trước (DEQUEUE). Người mới đến phải xếp CUỐI hàng (ENQUEUE). Ai đến trước được mua vé trước → đó chính là FIFO.

4.1. HTML Partial LÀ GÌ?

HTML partial là 1 file HTML nhỏ chứa 1 phần giao diện (vd: thanh tiêu đề, bảng dữ liệu, form...). Django dùng {% include %} để nhúng partial vào trang chính.

4.6. File goi_du_lieu.js — gọi API bằng Fetch

tep_tinh/ma_lenh/goi_du_lieu.js
/**
 * Gọi API GET /danh-sach để lấy toàn bộ danh sách mặt hàng.
 */
async function lay_danh_sach() {
    try {
        const phan_hoi = await fetch('/danh-sach');         // Gửi GET request
        const du_lieu  = await phan_hoi.json();             // Parse JSON
        return du_lieu;
    } catch (loi) {
        console.error('Lỗi khi lấy danh sách:', loi);
        hien_thong_bao('Không thể kết nối đến máy chủ.', 'loi');
        return null;
    }
}


/**
 * Gọi API POST /them-san-pham để thêm 1 sản phẩm mới.
 */
async function them_san_pham(du_lieu) {
    try {
        const phan_hoi = await fetch('/them-san-pham', {
            method:  'POST',
            headers: { 'Content-Type': 'application/json' },
            body:    JSON.stringify(du_lieu)                // Chuyển object → JSON string
        });
        const ket_qua = await phan_hoi.json();
        return { trang_thai: phan_hoi.status, du_lieu: ket_qua };
    } catch (loi) {
        console.error('Lỗi khi thêm sản phẩm:', loi);
        return null;
    }
}

Giải thích từng dòng (cho người mới học JavaScript):

  • async function — khai báo hàm bất đồng bộ. Cho phép dùng await.
  • await fetch('/danh-sach') — gửi HTTP GET request tới URL /danh-sach. await chờ response về.
  • await phan_hoi.json() — parse body response từ JSON string thành object JS.
  • try { ... } catch (loi) { ... } — bắt lỗi. Nếu fetch thất bại (mất mạng, server sập), code trong catch sẽ chạy.
  • console.error(...) — in lỗi ra console (F12 → tab Console).
  • fetch('/them-san-pham', { method: 'POST', ... }) — gửi POST request. headers khai báo loại nội dung, body chứa dữ liệu gửi đi.
  • JSON.stringify(du_lieu) — chuyển object JS thành JSON string để gửi đi.

Giải thích từng dòng:
