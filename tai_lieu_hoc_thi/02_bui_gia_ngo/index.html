BÀI 1 — NGĂN XẾP (STACK) — LIFO

1.1. Stack LÀ GÌ?

Ngăn xếp (Stack) là cấu trúc dữ liệu theo nguyên tắc LIFO — Last In, First Out (Vào sau, ra trước).

💡 Tưởng tượng đời thường: Hãy nghĩ về chồng đĩa trong nhà ăn. Bạn cất đĩa lên trên cùng chồng (push). Khi cần lấy đĩa, bạn lấy từ trên cùng (pop). Đĩa cất sau cùng sẽ được lấy ra đầu tiên — đó chính là LIFO.

1.2. So sánh với Queue (FIFO)

Đặc điểmStack (LIFO)Queue (FIFO)
Nguyên tắcVào sau, ra trướcVào trước, ra trước
Ví dụ đời thườngChồng đĩaXếp hàng mua vé
Số đầu vào/ra1 đầu (đỉnh)2 đầu (đầu + cuối)
Ứng dụng trong đồ ánUndo/RedoGiỏ hàng + thanh toán

1.4. Cài đặt trong đồ án (file models/ngan_xep.py)

1.4.1. Lớp NutNganXep (Node của Stack)

models/ngan_xep.py class NutNganXep: def __init__(self, du_lieu): self.du_lieu = du_lieu # Dữ liệu chứa trong mắt xích self.duoi = None # Trỏ tới mắt xích nằm ngay BÊN DƯỚI

Giải thích: Mỗi nút trong Stack chỉ cần 1 con trỏ duoi (trỏ xuống nút bên dưới). Khác với DSLK đôi (cần 2 con trỏ truoc + sau).

1.4.2. Lớp NganXep

models/ngan_xep.py class NganXep: def __init__(self): self.dinh = None # Phần tử trên cùng (None = rỗng) self.so_luong = 0 # Số phần tử đang có

Stack chỉ cần giữ con trỏ dinh (đỉnh). Từ đỉnh có thể đi xuống bằng duoi.

1.4.3. Hàm day_vao() — PUSH

def day_vao(self, du_lieu): nut_moi = NutNganXep(du_lieu) # 1. Tạo nút mới nut_moi.duoi = self.dinh # 2. Nút mới trỏ xuống nút đang ở đỉnh self.dinh = nut_moi # 3. Đỉnh mới = nút vừa tạo self.so_luong += 1

1.4.4. Hàm lay_ra() — POP

def lay_ra(self): if self.dinh is None: return None # Stack rỗng → trả None nut_lay = self.dinh # Lưu nút đỉnh self.dinh = self.dinh.duoi # Đỉnh mới = nút bên dưới self.so_luong -= 1 return nut_lay.du_lieu # Trả về dữ liệu

1.4.5. Hàm xem_dinh() — PEEK

def xem_dinh(self): if self.dinh is None: return None return self.dinh.du_lieu # Trả dữ liệu đỉnh, KHÔNG xóa

BÀI 2 — LƯU TRỮ DỮ LIỆU VỚI SQLITE

2.1. SQLite LÀ GÌ?

SQLite là hệ quản trị CSDL nhúng — không cần cài server riêng, chỉ cần 1 file .db trên đĩa. Python có sẵn thư viện sqlite3 trong bộ chuẩn — không cần cài thêm gì.

💡 So sánh: MySQL/PostgreSQL cần cài server riêng. SQLite chỉ là 1 file (quan_ly_nha_sach.db trong đồ án), dễ mang đi, dễ backup.

2.2. Các thao tác cơ bản (CRUD)

CRUD = Create + Read + Update + Delete

Thao tácCâu lệnh SQLTrong đồ án
Create (Tạo bảng)CREATE TABLE IF NOT EXISTS ...Khởi tạo bảng SanPham
Create (Thêm dòng)INSERT INTO ... VALUES (...)Thêm sản phẩm mới
Read (Đọc)SELECT * FROM ...Load sản phẩm khi mở app
Delete (Xóa)DELETE FROM ...Xóa hết trước khi ghi lại

2.3. Cấu trúc bảng SanPham trong đồ án

data/sqlite_db.py CREATE TABLE IF NOT EXISTS SanPham ( ma_so TEXT PRIMARY KEY, # Mã sản phẩm (khóa chính, không trùng) loai TEXT NOT NULL, # Loại: Sach/TapChi/Bao/LuanVan/BanThao ten TEXT NOT NULL, # Tên sản phẩm gia_co_ban REAL NOT NULL, # Giá cơ bản (số thực) so_luong INTEGER NOT NULL, # Số lượng tồn (số nguyên) thong_tin_1 TEXT, # Thuộc tính riêng 1 (vd: tác giả) thong_tin_2 TEXT # Thuộc tính riêng 2 (vd: NXB) )

Ý tưởng: Dùng 2 cột thong_tin_1 và thong_tin_2 để lưu thuộc tính riêng của từng loại (Sách có tác giả + NXB; Tạp chí chỉ có số phát hành...). Tránh phải tạo 5 bảng riêng.

2.4. Quy trình kết nối SQLite trong Python

import sqlite3 # 1. KẾT NỐI tới file .db (nếu chưa có sẽ tự tạo mới) conn = sqlite3.connect("data/quan_ly_nha_sach.db") # 2. TẠO CURSOR — con trỏ để thực thi câu lệnh SQL cursor = conn.cursor() # 3. THỰC THI câu lệnh SQL cursor.execute("SELECT * FROM SanPham") rows = cursor.fetchall() # Lấy tất cả dòng # 4. COMMIT (lưu thay đổi) — chỉ cần khi INSERT/UPDATE/DELETE conn.commit() # 5. ĐÓNG KẾT NỐI conn.close()
⚠️ Quan trọng: Phải conn.commit() sau khi INSERT/UPDATE/DELETE, nếu không dữ liệu KHÔNG được lưu. Quên commit() là lỗi rất hay gặp!

2.5. Hàm luu_du_lieu_sqlite() trong đồ án

data/sqlite_db.py def luu_du_lieu_sqlite(mang_san_pham): try: khoi_tao_db() # Tạo bảng nếu chưa có conn = sqlite3.connect(DB_PATH) cursor = conn.cursor() # Xóa dữ liệu cũ để đồng bộ cursor.execute("DELETE FROM SanPham") for sp in mang_san_pham: data = sp.to_dict() # Đa hình: chuyển sang dict loai = data["loai"] t1, t2 = "", "" # Lấy thuộc tính riêng theo loại if loai == "Sach": t1 = data.get("tac_gia", "") t2 = data.get("nha_xuat_ban", "") elif loai == "TapChi": t1 = data.get("so_phat_hanh", "") # ... (các loại khác tương tự) # INSERT dùng tham số ? để chống SQL Injection cursor.execute(''' INSERT INTO SanPham (ma_so, loai, ten, gia_co_ban, so_luong, thong_tin_1, thong_tin_2) VALUES (?, ?, ?, ?, ?, ?, ?) ''', (sp.ma_so, loai, sp.ten, sp.gia_co_ban, sp.so_luong, t1, t2)) conn.commit() conn.close() return True except Exception as loi: print(f"[LỖI] Không lưu được CSDL: {loi}") return False

Các điểm quan trọng:

  • try/except — bọc toàn bộ để nếu CSDL bị khóa/lỗi thì không crash chương trình.
  • Xóa hết rồi INSERT lại — cách đơn giản nhất để đồng bộ DSLK xuống DB. (Cách pro hơn là UPDATE, nhưng phức tạp hơn.)
  • Dùng ? thay vì nối chuỗi — chống SQL Injection (lỗi bảo mật nghiêm trọng).
  • data = sp.to_dict() — dùng đa hình để lấy đúng thuộc tính của từng loại sản phẩm.

2.6. Hàm doc_du_lieu_sqlite() — đọc + đa hình

def doc_du_lieu_sqlite(): try: danh_sach = [] conn = sqlite3.connect(DB_PATH) cursor = conn.cursor() cursor.execute("SELECT * FROM SanPham") rows = cursor.fetchall() for row in rows: ma_so, loai, ten, gia_co_ban, so_luong, t1, t2 = row # Dùng đa hình: tạo đúng lớp con theo `loai` if loai == "Sach": sp = Sach(ma_so, ten, gia_co_ban, so_luong, t1, t2) elif loai == "TapChi": sp = TapChi(ma_so, ten, gia_co_ban, so_luong, t1) # ... (các loại khác) danh_sach.append(sp) conn.close() return danh_sach except Exception as loi: print(f"[LỖI] {loi}") return []
💡 Vì sao quan trọng? Khi đọc từ SQLite, ta tạo đúng lớp con (Sach/TapChi/Bao/...) dựa vào cột loai. Nhờ vậy, đa hình tinh_gia_ban() vẫn hoạt động sau khi đọc từ CSDL.

BÀI 4 — KIỂM THỬ (TEST) — test_app.py

4.1. Vì sao phải test?

Test là chạy thử chương trình với dữ liệu mẫu để chứng minh code chạy đúng. Khi đi thi, thầy cô sẽ hỏi "em test chưa?" → mở file test_app.py ra là có câu trả lời.

4.2. Cấu trúc 1 test case

# 1. Chuẩn bị dữ liệu mẫu sp1 = Sach("S01", "Lập trình C++", 100000, 10, "Bjarne", "NXB Trẻ") # 2. Gọi hàm cần test gia = sp1.tinh_gia_ban() # 3. In ra để xem kết quả print(f"Giá bán Sách: {gia}") # 4. Kiểm tra kết quả đúng không (assert) assert gia == 120000, "Giá bán Sách phải = 100000 × 1.20 = 120000"

assert LÀ GÌ? Là câu lệnh Python: nếu điều kiện SAI thì ném lỗi và dừng chương trình. Dùng để kiểm tra kết quả test có đúng kỳ vọng không.
