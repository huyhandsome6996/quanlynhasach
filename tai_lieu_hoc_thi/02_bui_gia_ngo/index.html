BÀI 1 — NGĂN XẾP (STACK) — LIFO TÚ phụ trách

📌 Lưu ý: Stack/Queue do Tú phụ trách trong code. Nhưng Ngọ vẫn cần hiểu vì nó là cơ sở của tính năng Undo/Redo — và thi có thể hỏi chéo.

1.1. Stack LÀ GÌ?

Ngăn xếp (Stack) là cấu trúc dữ liệu theo nguyên tắc LIFO — Last In, First Out (Vào sau, ra trước).

💡 Tưởng tượng đời thường: Hãy nghĩ về chồng đĩa trong nhà ăn. Bạn cất đĩa lên trên cùng chồng (push). Khi cần lấy đĩa, bạn lấy từ trên cùng (pop). Đĩa cất sau cùng sẽ được lấy ra đầu tiên — đó chính là LIFO.

1.4.1. Lớp NodeNganXep (Node của Stack)

ung_dung_giao_tiep/loi_thuat_toan/cau_truc_du_lieu.py
class NodeNganXep:
    def __init__(self, data):
        self.data  = data      # Dữ liệu chứa trong mắt xích
        self.duoi  = None      # Trỏ tới mắt xích nằm ngay BÊN DƯỚI

Giải thích từng dòng:

  • class NodeNganXep: — khai báo lớp Node cho Stack.
  • def __init__(self, data): — hàm tạo, tự chạy khi tạo Node mới.
  • self.data = data — lưu dữ liệu vào thuộc tính data.
  • self.duoi = None — con trỏ trỏ tới Node bên dưới. Ban đầu None (chưa có).
  • Mỗi nút Stack chỉ cần 1 con trỏ duoi (trỏ xuống). Khác với DSLK đôi (cần 2 con trỏ truoc + sau).

1.4.2. Lớp NganXep

ung_dung_giao_tiep/loi_thuat_toan/cau_truc_du_lieu.py
class NganXep:
    def __init__(self):
        self.dinh      = None     # Phần tử trên cùng (None = rỗng)
        self.so_luong  = 0        # Số phần tử đang có

Stack chỉ cần giữ con trỏ dinh (đỉnh). Từ đỉnh có thể đi xuống bằng duoi.

1.4.3. Hàm day_vao() — PUSH

def day_vao(self, data):
    node_moi = NodeNganXep(data)    # 1. Tạo nút mới
    node_moi.duoi = self.dinh       # 2. Nút mới trỏ xuống nút đang ở đỉnh
    self.dinh = node_moi            # 3. Đỉnh mới = nút vừa tạo
    self.so_luong += 1

Giải thích từng dòng:

  • node_moi = NodeNganXep(data) — tạo Node mới chứa dữ liệu.
  • node_moi.duoi = self.dinh — Node mới trỏ xuống Node đang ở đỉnh. Như vậy Node mới nằm trên đỉnh cũ.
  • self.dinh = node_moi — cập nhật đỉnh = Node mới. Bước này rất quan trọng — nếu quên, Stack sẽ mất Node mới.
  • self.so_luong += 1 — tăng bộ đếm.

1.4.4. Hàm lay_ra() — POP

def lay_ra(self):
    if self.dinh is None:
        return None                # Stack rỗng → trả None
    data = self.dinh.data          # Lưu dữ liệu đỉnh
    self.dinh = self.dinh.duoi     # Đỉnh mới = nút bên dưới
    self.so_luong -= 1
    return data                    # Trả về dữ liệu

Giải thích từng dòng:

  • if self.dinh is None: — kiểm tra Stack rỗng. Tránh lỗi NoneType.
  • data = self.dinh.data — lưu dữ liệu đỉnh vào biến tạm. Phải làm TRƯỚC khi đổi self.dinh.
  • self.dinh = self.dinh.duoi — đỉnh mới = Node bên dưới. Node đỉnh cũ bị "bỏ rơi" (garbage collection sẽ dọn).
  • return data — trả dữ liệu đã lưu.

1.4.5. Hàm xem_dinh() — PEEK

def xem_dinh(self):
    if self.dinh is None:
        return None
    return self.dinh.data     # Trả dữ liệu đỉnh, KHÔNG xóa

Khác lay_ra() ở chỗ không giảm so_luong và không đổi self.dinh. Chỉ đọc.

BÀI 2 — LƯU TRỮ DỮ LIỆU VỚI SQLITE NGỌ

2.1. SQLite LÀ GÌ?

SQLite là hệ quản trị CSDL nhúng — không cần cài server riêng, chỉ cần 1 file .db trên đĩa. Python có sẵn thư viện sqlite3 trong bộ chuẩn — không cần cài thêm gì.

💡 So sánh: MySQL/PostgreSQL cần cài server riêng. SQLite chỉ là 1 file (db.sqlite3 trong đồ án), dễ mang đi, dễ backup.

2.2. Các thao tác cơ bản (CRUD)

CRUD = Create + Read + Update + Delete

2.4. Quy trình kết nối SQLite trong Python

import sqlite3

# 1. KẾT NỐI tới file .db (nếu chưa có sẽ tự tạo mới)
conn = sqlite3.connect("db.sqlite3")

# 2. TẠO CURSOR — con trỏ để thực thi câu lệnh SQL
cursor = conn.cursor()

# 3. THỰC THI câu lệnh SQL
cursor.execute("SELECT * FROM bang_mat_hang")
rows = cursor.fetchall()      # Lấy tất cả dòng

# 4. COMMIT (lưu thay đổi) — chỉ cần khi INSERT/UPDATE/DELETE
conn.commit()

# 5. ĐÓNG KẾT NỐI
conn.close()
⚠️ Quan trọng: Phải conn.commit() sau khi INSERT/UPDATE/DELETE, nếu không dữ liệu KHÔNG được lưu. Quên commit() là lỗi rất hay gặp!

2.5. Hàm tao_bang_neu_chua_co() — tạo bảng (try/except/finally)

ung_dung_giao_tiep/loi_thuat_toan/ket_noi_sqlite.py
def tao_bang_neu_chua_co():
    """Tạo bảng nếu chưa tồn tại. Áp dụng try-except-finally."""
    ket_noi = None
    try:
        duong_dan = lay_duong_dan_db()
        ket_noi = sqlite3.connect(duong_dan)        # Mở kết nối
        con_tro = ket_noi.cursor()
        con_tro.execute('''CREATE TABLE IF NOT EXISTS bang_mat_hang (
            ma_so TEXT PRIMARY KEY,
            loai_hang TEXT NOT NULL,
            ...
        )''')
        ket_noi.commit()                             # Lưu thay đổi
    except sqlite3.Error as loi:
        print(f'[LOI DATABASE] Loi khi tao bang: {str(loi)}')
        raise                                        # Ném lỗi lên để caller xử lý
    finally:
        if ket_noi:
            ket_noi.close()                          # Đảm bảo đóng kết nối

Giải thích try/except/finally (theo barem):

  • try: — khối thử. Code có thể gây lỗi nằm trong này.
  • except sqlite3.Error as loi: — bắt lỗi SQLite riêng (ví dụ: bảng đã tồn tại với cấu trúc khác).
  • raise — ném lỗi lên để caller (hàm gọi) biết và xử lý tiếp.
  • finally: — khối này luôn chạy dù có lỗi hay không. Đảm bảo ket_noi.close() luôn được gọi → tránh rò rỉ tài nguyên.
  • if ket_noi: — kiểm tra None trước khi close() (nếu connect() thất bại thì ket_noi vẫn là None).

def tai_du_lieu(danh_sach_lien_ket):

            elif loai_hang == 'Tạp chí':
                san_pham = TapChi(...)
            # ... (BaoGiay, LuanVan, BanThao tương tự)

Giải thích từng dòng:

  • con_tro.description — metadata của cột kết quả. Dùng để lấy tên cột.
  • [mo_ta[0] for mo_ta in con_tro.description] — list comprehension trích tên cột.
  • dict(zip(ten_cot, hang)) — ghép 2 list thành dict. Ví dụ: {'ma_so': 'S01', 'ten_san_pham': '...', ...}.
  • Đa hình: Tùy vào loai_hang mà tạo Sach(...), TapChi(...), ... Đây chính là đa hình OOP — cùng giao diện tinh_gia_ban() nhưng tính theo công thức riêng của từng loại.

2.7. Hàm luu_mot_san_pham() — INSERT với chống SQL Injection

ung_dung_giao_tiep/loi_thuat_toan/ket_noi_sqlite.py
def luu_mot_san_pham(san_pham):
    """Lưu 1 sản phẩm mới vào SQLite. Chống SQL Injection bằng tham số ?."""
    ket_noi = None
    try:
        duong_dan = lay_duong_dan_db()
        ket_noi = sqlite3.connect(duong_dan)
        con_tro = ket_noi.cursor()

        # Dùng ? thay vì nối chuỗi — chống SQL Injection
        con_tro.execute('''
            INSERT INTO bang_mat_hang
                (ma_so, loai_hang, ten_san_pham, gia_co_ban, ton_kho, tac_gia, nha_xuat_ban, ...)
            VALUES (?, ?, ?, ?, ?, ?, ?, ...)
        ''', (
            san_pham.ma_so,
            san_pham.loai_hang,
            san_pham.ten_san_pham,
            san_pham.gia_co_ban,
            san_pham.ton_kho,
            getattr(san_pham, 'tac_gia', None),
            getattr(san_pham, 'nha_xuat_ban', None),
            ...
        ))
        ket_noi.commit()
    except sqlite3.Error as loi:
        print(f'[LOI] {str(loi)}')
        raise
    finally:
        if ket_noi:
            ket_noi.close()

Tại sao dùng ? thay vì nối chuỗi?

  • SAI (nguy hiểm): f"INSERT INTO ... VALUES ('{san_pham.ma_so}', ...)" — nếu user nhập mã '); DROP TABLE bang_mat_hang;-- → bảng bị xóa!
  • ĐÚNG (an toàn): con_tro.execute('... VALUES (?, ...)', (san_pham.ma_so, ...)) — SQLite tự escape ký tự đặc biệt.
  • getattr(san_pham, 'tac_gia', None) — lấy thuộc tính tac_gia, nếu không có trả None. Tránh lỗi AttributeError khi sản phẩm không có tac_gia (vd: Báo giấy).

class NguoiDung:
    """Lớp đại diện cho 1 tài khoản đăng nhập hệ thống."""

    def __init__(self, ten_dang_nhap, mat_khau, ho_ten, vai_tro='nhan_vien'):
        self.ten_dang_nhap = ten_dang_nhap      # Tên đăng nhập (duy nhất)
        self.mat_khau      = mat_khau           # Mật khẩu (plaintext cho đơn giản)
        self.ho_ten        = ho_ten             # Họ tên để hiển thị
        self.vai_tro       = vai_tro            # 'admin' hoặc 'nhan_vien'

    def la_admin(self):
        """Kiểm tra người dùng có phải admin không."""
        return self.vai_tro == 'admin'

    def chuyen_thanh_dict(self):
        """Chuyển thành dict để trả về JSON (KHÔNG trả mật khẩu)."""
        return {
            'ten_dang_nhap': self.ten_dang_nhap,
            'ho_ten':        self.ho_ten,
            'vai_tro':       self.vai_tro,
            'la_admin':      self.la_admin()
        }

Giải thích từng dòng:

  • def __init__(self, ten_dang_nhap, mat_khau, ho_ten, vai_tro='nhan_vien'): — hàm tạo với 4 tham số. vai_tro='nhan_vien' là giá trị mặc định — nếu không truyền vào thì mặc định là nhân viên.
  • self.ten_dang_nhap = ten_dang_nhap — gán tham số vào thuộc tính đối tượng.
  • def la_admin(self): — phương thức kiểm tra quyền. Trả True nếu vai_tro == 'admin', ngược lại False.
  • def chuyen_thanh_dict(self): — chuyển đối tượng thành dict (cấu trúc dữ liệu JSON tương đương). Quan trọng: không trả mat_khau để bảo mật.

3.4. Hàm kiem_tra_dang_nhap() — kiểm tra tài khoản

def kiem_tra_dang_nhap(ten_dang_nhap, mat_khau):
    """Kiểm tra thông tin đăng nhập. Trả về NguoiDung nếu đúng, None nếu sai."""
    try:
        from .ket_noi_sqlite import xac_thuc_nguoi_dung
        thong_tin = xac_thuc_nguoi_dung(ten_dang_nhap)   # SELECT từ SQLite
        if thong_tin is None:
            return None                                  # Sai tên đăng nhập
        if thong_tin['mat_khau_hash'] != mat_khau:
            return None                                  # Sai mật khẩu
        # Đúng cả 2 → tạo đối tượng NguoiDung trả về
        return NguoiDung(
            ten_dang_nhap=thong_tin['ten_dang_nhap'],
            mat_khau=thong_tin['mat_khau_hash'],
            ho_ten=thong_tin['ho_ten'],
            vai_tro=thong_tin['vai_tro']
        )
    except Exception as loi:
        print(f'[LOI] Loi khi kiem tra dang nhap: {str(loi)}')
        return None

Giải thích từng dòng:

  • xac_thuc_nguoi_dung(ten_dang_nhap) — SELECT từ SQLite theo tên đăng nhập. Trả dict nếu tìm thấy, None nếu không.
  • if thong_tin is None: — tên đăng nhập không tồn tại.
  • if thong_tin['mat_khau_hash'] != mat_khau: — mật khẩu không khớp. Lưu ý: so sánh chuỗi plaintext chỉ dành cho học, thực tế phải băm bằng bcrypt/hashlib.
  • Nếu qua 2 bước kiểm tra → tạo NguoiDung từ thông tin SQLite và trả về.

4.1. Vì sao phải test?

Test là chạy thử chương trình với dữ liệu mẫu để chứng minh code chạy đúng. Khi đi thi, thầy cô sẽ hỏi "em test chưa?" → mở file kiem_thu.py ra là có câu trả lời.

🚗 Ví dụ đời thường: Kiểm tra xe trước khi giao cho khách

Hãy tưởng tượng bạn làm tại xưởng sản xuất ô tô. Trước khi giao 1 chiếc xe cho khách, bạn không thể nhìn bề ngoài rồi bảo "ok ngon rồi". Bạn phải:

  • 🛞 Test lốp — bơm hơi, đạp thử xem có xẹp không
  • 🛢 Test động cơ — nổ máy, nghe tiếng, đo áp suất dầu
  • 💡 Test đèn — bật/tắt từng đèn, kiểm tra cháy bóng
  • 🛑 Test phanh — chạy thử 30km/h rồi đạp phanh gấp

Lập trình cũng vậy! Khi viết xong 1 hàm, bạn không thể bảo "nó chạy được rồi" mà phải chạy thử với dữ liệu cụ thể, kiểm tra kết quả có đúng kỳ vọng không. Đó chính là test case.

from .danh_sach_lien_ket import Node, DoublyLinkedList

class MatHangGia:
    """Lớp giả lập mặt hàng để test DSLK."""
    def __init__(self, ma_so, ten_san_pham, gia_co_ban):
        self.ma_so = ma_so
        self.ten_san_pham = ten_san_pham
        self.gia_co_ban = gia_co_ban

    def tinh_gia_ban(self):
        return self.gia_co_ban

    def chuyen_thanh_dict(self):
        return {'ma_so': self.ma_so,
                'ten_san_pham': self.ten_san_pham,
                'gia_co_ban': self.gia_co_ban,
                'gia_ban': self.tinh_gia_ban()}

def tao_du_lieu_gia():
    """Tạo 5 sản phẩm mẫu để test."""
    ds = DoublyLinkedList()
    ds.them_vao_cuoi(MatHangGia('MH001', 'Sách Lập trình Python',   120000))
    ds.them_vao_cuoi(MatHangGia('MH002', 'Tạp chí Khoa học',         45000))
    ds.them_vao_cuoi(MatHangGia('MH003', 'Báo Nhân dân',              8000))
    ds.them_vao_cuoi(MatHangGia('MH004', 'Sách Cấu trúc dữ liệu',    95000))
    ds.them_vao_cuoi(MatHangGia('MH005', 'Tạp chí Công nghệ',        55000))
    return ds

def kiem_thu_co_ban():
    """Chạy 6 test case: thêm, duyệt, xóa, tìm, sắp xếp."""
    print('=' * 60)
    print('KIỂM THỬ DANH SÁCH LIÊN KẾT ĐÔI')
    print('=' * 60)

    # [1] Test THÊM
    ds = tao_du_lieu_gia()
    print(f'\n[1] Thêm phần tử: Số lượng = {ds.so_luong}')
    assert ds.so_luong == 5, 'Phải có 5 phần tử'

    # [2] Test CHUYỂN THÀNH DANH SÁCH
    danh_sach = ds.chuyen_thanh_danh_sach()
    print(f'\n[2] Chuyển thành danh sách dict: {danh_sach}')
    assert len(danh_sach) == 5

    # [3] Test XÓA
    ket_qua_xoa = ds.xoa_node('MH003')
    print(f"\n[3] Xóa MH003: {('Thành công' if ket_qua_xoa else 'Thất bại')}")
    assert ket_qua_xoa == True
    assert ds.so_luong == 4

    # [4] Test XÓA mã không tồn tại
    ket_qua_xoa_2 = ds.xoa_node('MH999')
    print(f"\n[4] Xóa MH999 (không tồn tại): {ket_qua_xoa_2}")
    assert ket_qua_xoa_2 == False
    assert ds.so_luong == 4   # Không đổi

    # [5] Test TÌM KIẾM
    ds2 = tao_du_lieu_gia()
    ket_qua_tim = ds2.tim_kiem('sách')
    print(f"\n[5] Tìm kiếm 'sách': Tìm thấy {len(ket_qua_tim)} kết quả")
    assert len(ket_qua_tim) == 2   # 'Sách Lập trình' + 'Sách Cấu trúc'

    # [6] Test SẮP XẾP
    ds3 = tao_du_lieu_gia()
    print(f'\n[6] Sắp xếp theo giá bán:')
    print(f'    Trước: {ds3}')
    ds3.sap_xep_tron('gia_ban')
    print(f'    Sau:   {ds3}')

    print('\n' + '=' * 60)
    print('HOÀN THÀNH KIỂM THỬ!')
    print('=' * 60)

if __name__ == '__main__':
    kiem_thu_co_ban()

Giải thích từng phần:

  • class MatHangGia: — lớp giả lập (mock) mặt hàng, chỉ có ma_so, ten_san_pham, gia_co_ban. Đơn giản hơn lớp thật để test dễ.
  • def tao_du_lieu_gia(): — hàm tiện ích, tạo 5 sản phẩm mẫu. Dùng nhiều lần trong các test case → tránh lặp code.
  • assert ds.so_luong == 5, 'Phải có 5 phần tử' — kiểm tra số lượng. Nếu sai → dừng và in thông báo.
  • if __name__ == '__main__': — chỉ chạy kiem_thu_co_ban() khi file này được chạy trực tiếp (python kiem_thu.py), không chạy khi import.
