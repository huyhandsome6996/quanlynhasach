BÀI 1 — DANH SÁCH LIÊN KẾT ĐÔI (Doubly Linked List)

1.1. Danh sách liên kết đôi LÀ GÌ?

Danh sách liên kết đôi (Doubly Linked List) là một cấu trúc dữ liệu gồm nhiều nút (node) nối với nhau bằng con trỏ. Mỗi nút chứa:

  • du_lieu — dữ liệu thực tế (vd: 1 sản phẩm)
  • truoc — con trỏ trỏ tới nút đứng trước nó
  • sau — con trỏ trỏ tới nút đứng sau nó
💡 Tưởng tượng đời thường: Hãy tưởng tượng một đoàn người xếp hàng. Mỗi người biết người đứng trước mình là ai và người đứng sau mình là ai. Người đầu tiên không có ai trước, người cuối không có ai sau.

Vì sao đồ án dùng DSLK đôi? Vì cửa hàng sách thường xuyên THÊM / XÓA / SỬA sản phẩm. Dùng DSLK đôi thì các thao tác này rất nhanh, không phải dời dữ liệu như mảng.

1.3. Cài đặt trong đồ án (file models/linked_list.py)

1.3.1. Lớp Nut (Node) — một mắt xích

models/linked_list.py # Cài đặt cấu trúc Node cho Danh sách liên kết đôi class Nut: def __init__(self, du_lieu): self.du_lieu = du_lieu # Dữ liệu mà nút này chứa self.truoc = None # Con trỏ trỏ về nút trước (chưa có gì) self.sau = None # Con trỏ trỏ tới nút sau (chưa có gì)

Giải thích từng dòng:

  • class Nut: — khai báo 1 lớp (class) tên là Nut. Trong Python dùng từ khóa class.
  • def __init__(self, du_lieu): — đây là hàm tạo (constructor). Tự động chạy khi tạo Nut(...). self là từ khóa bắt buộc ở tham số đầu — nó đại diện cho chính đối tượng đang được tạo.
  • self.du_lieu = du_lieu — gán giá trị tham số du_lieu vào thuộc tính du_lieu của đối tượng.
  • self.truoc = None — khởi tạo con trỏ truoc bằng None (rỗng). Khi mới tạo, nút chưa liên kết với nút nào.
  • self.sau = None — tương tự, con trỏ sau ban đầu cũng rỗng.

1.3.2. Lớp DanhSachLienKetDoi

models/linked_list.py class DanhSachLienKetDoi: def __init__(self): self.dau = None # Head — con trỏ tới nút ĐẦU TIÊN self.cuoi = None # Tail — con trỏ tới nút CUỐI CÙNG self.so_luong = 0 # Đếm số nút đang có

Lớp này chỉ cần giữ 3 thông tin: đầu, cuối, số lượng. Từ dau ta có thể đi đến bất kỳ nút nào bằng cách nhảy theo sau. Từ cuoi ta có thể đi ngược lại bằng truoc.

1.3.3. Phương thức them_vao_cuoi() — thêm 1 nút vào cuối danh sách

models/linked_list.py def them_vao_cuoi(self, du_lieu): nut_moi = Nut(du_lieu) # 1. Tạo nút mới chứa dữ liệu if self.dau is None: # 2. Nếu danh sách đang rỗng self.dau = nut_moi # → nút mới là vừa đầu vừa cuối self.cuoi = nut_moi else: # 3. Nếu đã có nút khác self.cuoi.sau = nut_moi # → nút cuối hiện tại trỏ tới nút mới nut_moi.truoc = self.cuoi # → nút mới trỏ ngược về nút cuối self.cuoi = nut_moi # → cập nhật lại con trỏ `cuoi` self.so_luong += 1 # 4. Tăng bộ đếm

Giải thích:

  • Tạo nut_moi chứa S02 (ban đầu truoc=None, sau=None)
  • self.cuoi.sau = nut_moi — nút S01 (đang là cuoi) trỏ sau tới S02
  • nut_moi.truoc = self.cuoi — S02 trỏ truoc về S01
  • self.cuoi = nut_moi — cập nhật cuoi giờ là S02

1.3.4. Phương thức xoa_nut_theo_ma() — xóa 1 nút

models/linked_list.py def xoa_nut_theo_ma(self, ma_so): hien_tai = self.dau # Bắt đầu từ nút đầu tiên while hien_tai is not None: # Duyệt qua từng nút if hien_tai.du_lieu.ma_so == ma_so: # Tìm thấy nút cần xóa # CẬP NHẬT CON TRỎ CỦA NÚT TRƯỚC & SAU ĐỂ "BỎ QUA" NÚT HIỆN TẠI if hien_tai.truoc is not None: hien_tai.truoc.sau = hien_tai.sau # Nút trước trỏ sang nút sau else: self.dau = hien_tai.sau # Nếu là nút đầu → cập nhật dau if hien_tai.sau is not None: hien_tai.sau.truoc = hien_tai.truoc # Nút sau trỏ ngược về nút trước else: self.cuoi = hien_tai.truoc # Nếu là nút cuối → cập nhật cuoi self.so_luong -= 1 return True # Xóa xong, thoát hàm hien_tai = hien_tai.sau # Chưa thấy → đi tới nút tiếp theo return False # Không tìm thấy → trả False
💡 Ý tưởng cốt lõi: Để xóa 1 nút trong DSLK đôi, ta KHÔNG xóa vật lý nút đó, mà cho nút trước và nút sau của nó "bỏ qua" nó (trỏ thẳng tới nhau). Nút bị bỏ qua sẽ tự động mất khỏi danh sách.

BÀI 2 — THUẬT TOÁN MERGE SORT TRÊN DSLK

2.1. Merge Sort LÀ GÌ?

Merge Sort (sắp xếp trộn) là thuật toán sắp xếp dùng phương pháp chia để trị (divide and conquer). Ý tưởng gồm 3 bước:

  1. CHIA — Cắt danh sách làm 2 nửa
  2. TRỘN — Sắp xếp từng nửa (gọi đệ quy)
  3. GỘP — Trộn 2 nửa đã sắp xếp lại thành 1 danh sách đã sắp xếp
💡 Ví dụ đời thường: Bạn có 1 chồng bài kiểm tra lộn xộn. Thay vì sắp xếp cả chồng, bạn cắt làm 2 chồng nhỏ, nhờ 2 người bạn sắp xếp từng chồng, rồi bạn trộn 2 chồng đã sắp xếp lại với nhau. Nhanh hơn nhiều!

2.2. Độ phức tạp

Trường hợpĐộ phức tạpGhi chú
Tốt nhấtO(n log n)Luôn như nhau
Trung bìnhO(n log n)Ổn định, không phụ thuộc dữ liệu
Xấu nhấtO(n log n)Khác Bubble Sort (O(n²))

2.3. Kỹ thuật 2 con trỏ (nhanh / chậm) để tìm điểm giữa

Để cắt danh sách làm 2, ta cần tìm điểm giữa. Cách dễ nhất là đếm số nút rồi đi nửa đường. Nhưng có cách khéo hơn: dùng 2 con trỏ:

  • nhanh — đi 2 bước mỗi lần
  • cham — đi 1 bước mỗi lần

Khi nhanh đi hết danh sách thì cham đang ở giữa!

models/linked_list.py def _chia_doi(self, dau): nhanh = dau # Con trỏ nhanh đi 2 bước/lần cham = dau # Con trỏ chậm đi 1 bước/lần while nhanh.sau is not None and nhanh.sau.sau is not None: nhanh = nhanh.sau.sau # Nhanh đi 2 bước cham = cham.sau # Chậm đi 1 bước giua = cham.sau # Điểm giữa = nút ngay sau `cham` cham.sau = None # Cắt đôi: chấm dứt nửa đầu return giua # Trả về đầu của nửa sau

2.4. Hàm đệ quy _merge_sort()

models/linked_list.py def _merge_sort(self, dau, tieu_chi, tang_dan): # Điều kiện dừng đệ quy: 0 hoặc 1 nút thì đã sắp xếp rồi if dau is None or dau.sau is None: return dau # 1. CHIA — Cắt đôi danh sách giua = self._chia_doi(dau) # 2. GỌI ĐỆ QUY — Sắp xếp từng nửa trai = self._merge_sort(dau, tieu_chi, tang_dan) phai = self._merge_sort(giua, tieu_chi, tang_dan) # 3. TRỘN — Gộp 2 nửa đã sắp xếp return self._tron(trai, phai, tieu_chi, tang_dan)
⚠️ Hiểu về đệ quy (recursion): Hàm _merge_sort gọi chính nó với input nhỏ hơn. Quá trình cứ vậy cho đến khi điều kiện dừng (dau is None hoặc chỉ còn 1 nút). Khi đó hàm trả về danh sách đã sắp xếp (1 nút thì coi như đã sắp xếp). Các lời gọi đệ quy sẽ dần dần trộn các phần nhỏ lại thành kết quả cuối.

2.5. Hàm trộn _tron()

models/linked_list.py def _tron(self, trai, phai, tieu_chi, tang_dan): if trai is None: return phai if phai is None: return trai # Lấy giá trị để so sánh bằng hàm getattr (lấy thuộc tính động) gt_trai = getattr(trai.du_lieu, tieu_chi) gt_phai = getattr(phai.du_lieu, tieu_chi) # Tránh lỗi phân biệt chữ hoa/thường khi sắp xếp theo tên if isinstance(gt_trai, str): gt_trai = gt_trai.lower() if isinstance(gt_phai, str): gt_phai = gt_phai.lower() # Quyết định nút nào nhỏ hơn → đặt trước if tang_dan: dk = gt_trai <= gt_phai else: dk = gt_trai >= gt_phai if dk: ket_qua = trai ket_qua.sau = self._tron(trai.sau, phai, tieu_chi, tang_dan) if ket_qua.sau: ket_qua.sau.truoc = ket_qua ket_qua.truoc = None return ket_qua else: ket_qua = phai ket_qua.sau = self._tron(trai, phai.sau, tieu_chi, tang_dan) if ket_qua.sau: ket_qua.sau.truoc = ket_qua ket_qua.truoc = None return ket_qua

Giải thích chi tiết:

  • getattr(trai.du_lieu, tieu_chi) — lấy thuộc tính tieu_chi từ đối tượng trai.du_lieu. Nếu tieu_chi="gia_co_ban" thì相当于 trai.du_lieu.gia_co_ban.
  • isinstance(gt_trai, str) — kiểm tra xem gt_trai có phải chuỗi không. Nếu chuỗi thì .lower() để "ABC" và "abc" bằng nhau khi sắp xếp.
  • Phần if dk: ... else: ... — chọn nút nhỏ hơn (hoặc lớn hơn nếu giảm dần) để đặt làm ket_qua, rồi đệ quy trộn phần còn lại.
  • ket_qua.sau.truoc = ket_qua — cập nhật ngược con trỏ truoc để giữ tính chất "đôi" của DSLK.

3.1. PyQt6 LÀ GÌ?

PyQt6 là thư viện Python giúp tạo ra ứng dụng desktop có giao diện đồ họa (GUI) — tức là có cửa sổ, nút bấm, ô nhập, bảng... giống Word, Excel, Chrome. PyQt6 là phiên bản Python của framework Qt (C++).

💡 So sánh: Code Python thường chạy trong terminal (chỉ có chữ). Dùng PyQt6 thì có cửa sổ đồ họa với nút bấm, ô nhập — giống phần mềm thật. Người dùng không cần biết code vẫn dùng được.

3.2. Cài đặt PyQt6

terminal pip install PyQt6

3.3. Cấu trúc 1 chương trình PyQt6 tối giản

main.py (tối giản) import sys from PyQt6.QtWidgets import QApplication, QMainWindow app = QApplication(sys.argv) # 1. TẠO APP (bắt buộc) cua_so = QMainWindow() # 2. TẠO CỬA SỔ CHÍNH cua_so.setWindowTitle("Chương trình đầu tiên") cua_so.resize(800, 600) cua_so.show() # 3. HIỆN CỬA SỔ LÊN MÀN HÌNH sys.exit(app.exec()) # 4. CHẠY VÒNG LẶP SỰ KIỆN

Giải thích từng dòng:

  • import sys — nhập thư viện hệ thống (cần để lấy tham số command-line và thoát chương trình).
  • from PyQt6.QtWidgets import QApplication, QMainWindow — nhập 2 lớp cần thiết: QApplication (quản lý toàn app) và QMainWindow (cửa sổ chính).
  • app = QApplication(sys.argv) — bắt buộc phải có 1 instance QApplication. Nó quản lý vòng lặp sự kiện, style, font... Tham số sys.argv cho phép nhận tham số command-line.
  • cua_so = QMainWindow() — tạo 1 cửa sổ chính. QMainWindow có sẵn thanh menu, thanh trạng thái, vùng widget trung tâm.
  • cua_so.setWindowTitle(...) — đặt tiêu đề cửa sổ (hiện trên thanh tiêu đề).
  • cua_so.resize(800, 600) — đặt kích thước cửa sổ (rộng 800px, cao 600px).
  • cua_so.show() — bắt buộc. Mặc định cửa sổ ẩn, phải gọi .show() mới hiện.
  • sys.exit(app.exec()) — khởi động vòng lặp sự kiện. Hàm app.exec() sẽ chặn chương trình kết thúc, lắng nghe sự kiện (click, gõ phím...) cho đến khi user đóng cửa sổ. sys.exit() đảm bảo dọn dẹp bộ nhớ khi thoát.

4.1. Khai báo lớp và hàm tạo

ui/main_window.py class CuaSoChinh(QMainWindow): def __init__(self, nguoi_dung_hien_tai=None): super().__init__() # Gọi __init__ của QMainWindow self.setWindowTitle("Hệ Thống Quản Lý Cửa Hàng Sách") self.resize(1100, 700) # Thông tin người dùng đăng nhập self.nguoi_dung = nguoi_dung_hien_tai # 4 cấu trúc dữ liệu chính self.danh_sach = DanhSachLienKetDoi() # 1. Giải thích từng dòng:

  • class CuaSoChinh(QMainWindow): — khai báo lớp CuaSoChinh kế thừa từ QMainWindow. Nghĩa là CuaSoChinh sẽ có sẵn mọi tính năng của QMainWindow (tiêu đề, menu, status bar...) cộng thêm những gì ta viết.
  • def __init__(self, nguoi_dung_hien_tai=None): — hàm tạo. Tham số nguoi_dung_hien_tai mặc định None (nếu không truyền thì là None).
  • super().__init__() — gọi hàm tạo của lớp cha (QMainWindow). Bắt buộc để QMainWindow khởi tạo các thành phần nội bộ của nó.
  • self.nguoi_dung = nguoi_dung_hien_tai — lưu thông tin người dùng vào thuộc tính nguoi_dung để dùng trong cả lớp.
  • 4 dòng cấu trúc dữ liệu: khởi tạo 4 cấu trúc chính. Đây là xương sống của chương trình.
  • self.thiet_lap_giao_dien() — gọi hàm tạo giao diện (4.2).
  • self.doc_du_lieu_tu_file() — đọc dữ liệu từ SQLite nạp vào DSLK (4.5).

BÀI 5 — UNDO/REDO BẰNG 2 STACK (LIFO)

5.1. Ý tưởng

Undo/Redo dùng 2 Stack:

  • Stack Undo — lưu các thao tác đã làm. Bấm Undo → lấy 1 phần tử ra.
  • Stack Redo — lưu các thao tác đã Undo. Bấm Redo → lấy 1 phần tử ra.
💡 Quy tắc vàng: Mỗi lần user làm thao tác mới → xóa sạch Stack Redo.

5.2. Cấu trúc 1 "thao tác" trong Stack

# Thao tác THÊM {"thao_tac": "them", "sp": sp_moi} # Thao tác XÓA {"thao_tac": "xoa", "sp": sp_bi_xoa} # Thao tác SỬA {"thao_tac": "sua", "sp_cu": sp_cu, "sp_moi": sp_moi}

5.3. Hàm xu_ly_hoan_tac() — Undo

def xu_ly_hoan_tac(self): thao_tac = self.ngan_xep_hoan_tac.lay_ra() # Lấy từ đỉnh Stack Undo kieu = thao_tac["thao_tac"] if kieu == "them": # Đã THÊM → Undo = XÓA self.danh_sach.xoa_nut_theo_ma(thao_tac["sp"].ma_so) elif kieu == "xoa": # Đã XÓA → Undo = THÊM lại self.danh_sach.them_vao_cuoi(thao_tac["sp"]) elif kieu == "sua": # Đã SỬA → Undo = khôi phục sp_cu self.danh_sach.xoa_nut_theo_ma(thao_tac["sp_moi"].ma_so) self.danh_sach.them_vao_cuoi(thao_tac["sp_cu"]) self.ngan_xep_lam_lai.day_vao(thao_tac) # Đẩy sang Redo self.luu_du_lieu_ra_file() self.cap_nhat_bang_hien_thi()
