BÀI 1 — DANH SÁCH LIÊN KẾT ĐÔI (Doubly Linked List) HUY

1.1. Danh sách liên kết đôi LÀ GÌ?

Danh sách liên kết đôi (Doubly Linked List) là một cấu trúc dữ liệu gồm nhiều nút (node) nối với nhau bằng con trỏ. Mỗi nút chứa:

  • du_lieu — dữ liệu thực tế (vd: 1 sản phẩm)
  • truoc — con trỏ trỏ tới nút đứng trước nó
  • sau — con trỏ trỏ tới nút đứng sau nó
💡 Tưởng tượng đời thường: Hãy tưởng tượng một đoàn người xếp hàng. Mỗi người biết người đứng trước mình là ai và người đứng sau mình là ai. Người đầu tiên không có ai trước, người cuối không có ai sau.

1.2. So sánh với mảng (list) Python thường

Đặc điểm Mảng Python (list) Danh sách liên kết đôi
Cấp phát bộ nhớ Liên tục Rải rác (mỗi node 1 chỗ)
Truy cập theo chỉ số [i] O(1) — nhanh O(n) — phải duyệt từ đầu
Thêm/xóa ở đầu/cuối O(n) — phải dời cả mảng O(1) — chỉ đổi con trỏ
Dung lượng Cố định (hoặc copy khi full) Động (cần thêm thì cấp thêm)
Vì sao đồ án dùng DSLK đôi? Vì cửa hàng sách thường xuyên THÊM / XÓA / SỬA sản phẩm. Dùng DSLK đôi thì các thao tác này rất nhanh, không phải dời dữ liệu như mảng.

1.3. Cài đặt trong đồ án (file danh_sach_lien_ket.py)

1.3.1. Lớp Node — một mắt xích

ung_dung_giao_tiep/loi_thuat_toan/danh_sach_lien_ket.py
class Node:
    def __init__(self, du_lieu):
        self.du_lieu = du_lieu      # Dữ liệu mà nút này chứa
        self.truoc   = None         # Con trỏ trỏ về nút trước (chưa có gì)
        self.sau     = None         # Con trỏ trỏ tới nút sau (chưa có gì)

Giải thích từng dòng:

  • class Node: — khai báo 1 lớp (class) tên là Node. Trong Python dùng từ khóa class.
  • def __init__(self, du_lieu): — đây là hàm tạo (constructor). Tự động chạy khi tạo Node(...). self là từ khóa bắt buộc ở tham số đầu — nó đại diện cho chính đối tượng đang được tạo.
  • self.du_lieu = du_lieu — gán giá trị tham số du_lieu vào thuộc tính du_lieu của đối tượng.
  • self.truoc = None — khởi tạo con trỏ truoc bằng None (rỗng). Khi mới tạo, nút chưa liên kết với nút nào.
  • self.sau = None — tương tự, con trỏ sau ban đầu cũng rỗng.

1.3.2. Lớp DoublyLinkedList

ung_dung_giao_tiep/loi_thuat_toan/danh_sach_lien_ket.py
class DoublyLinkedList:
    def __init__(self):
        self.dau       = None       # Head — con trỏ tới nút ĐẦU TIÊN
        self.cuoi      = None       # Tail — con trỏ tới nút CUỐI CÙNG
        self.so_luong  = 0          # Đếm số nút đang có

Lớp này chỉ cần giữ 3 thông tin: đầu, cuối, số lượng. Từ dau ta có thể đi đến bất kỳ nút nào bằng cách nhảy theo sau. Từ cuoi ta có thể đi ngược lại bằng truoc.

1.3.3. Phương thức them_vao_cuoi() — thêm 1 nút vào cuối danh sách

ung_dung_giao_tiep/loi_thuat_toan/danh_sach_lien_ket.py
def them_vao_cuoi(self, du_lieu):
    node_moi = Node(du_lieu)              # 1. Tạo nút mới chứa dữ liệu
    if self.dau is None:                  # 2. Nếu danh sách đang rỗng
        self.dau  = node_moi              #    → nút mới là vừa đầu vừa cuối
        self.cuoi = node_moi
    else:                                 # 3. Nếu đã có nút khác
        self.cuoi.sau   = node_moi        #    → nút cuối hiện tại trỏ tới nút mới
        node_moi.truoc  = self.cuoi       #    → nút mới trỏ ngược về nút cuối
        self.cuoi       = node_moi        #    → cập nhật lại con trỏ `cuoi`
    self.so_luong += 1                    # 4. Tăng bộ đếm

Sau khi gọi them_vao_cuoi(S02):

Giải thích:

  • Tạo node_moi chứa S02 (ban đầu truoc=None, sau=None)
  • self.cuoi.sau = node_moi — nút S01 (đang là cuoi) trỏ sau tới S02
  • node_moi.truoc = self.cuoi — S02 trỏ truoc về S01
  • self.cuoi = node_moi — cập nhật cuoi giờ là S02

1.3.4. Phương thức xoa_node() — xóa 1 nút

ung_dung_giao_tiep/loi_thuat_toan/danh_sach_lien_ket.py
def xoa_node(self, ma_so):
    hien_tai = self.dau                       # Bắt đầu từ nút đầu tiên
    while hien_tai is not None:               # Duyệt qua từng nút
        if hien_tai.du_lieu.ma_so == ma_so:   # Tìm thấy nút cần xóa
            # CẬP NHẬT CON TRỎ CỦA NÚT TRƯỚC & SAU ĐỂ "BỎ QUA" NÚT HIỆN TẠI
            if hien_tai.truoc is not None:
                hien_tai.truoc.sau = hien_tai.sau    # Nút trước trỏ sang nút sau
            else:
                self.dau = hien_tai.sau                # Nếu là nút đầu → cập nhật dau

            if hien_tai.sau is not None:
                hien_tai.sau.truoc = hien_tai.truoc    # Nút sau trỏ ngược về nút trước
            else:
                self.cuoi = hien_tai.truoc              # Nếu là nút cuối → cập nhật cuoi

            self.so_luong -= 1
            return True                       # Xóa xong, thoát hàm
        hien_tai = hien_tai.sau                # Chưa thấy → đi tới nút tiếp theo
    return False                              # Không tìm thấy → trả False
💡 Ý tưởng cốt lõi: Để xóa 1 nút trong DSLK đôi, ta KHÔNG xóa vật lý nút đó, mà cho nút trước và nút sau của nó "bỏ qua" nó (trỏ thẳng tới nhau). Nút bị bỏ qua sẽ tự động mất khỏi danh sách.

BÀI 2 — THUẬT TOÁN MERGE SORT TRÊN DSLK HUY

2.1. Merge Sort LÀ GÌ?

Merge Sort (sắp xếp trộn) là thuật toán sắp xếp dùng phương pháp chia để trị (divide and conquer). Ý tưởng gồm 3 bước:

  1. CHIA — Cắt danh sách làm 2 nửa
  2. TRỘN — Sắp xếp từng nửa (gọi đệ quy)
  3. GỘP — Trộn 2 nửa đã sắp xếp lại thành 1 danh sách đã sắp xếp
💡 Ví dụ đời thường: Bạn có 1 chồng bài kiểm tra lộn xộn. Thay vì sắp xếp cả chồng, bạn cắt làm 2 chồng nhỏ, nhờ 2 người bạn sắp xếp từng chồng, rồi bạn trộn 2 chồng đã sắp xếp lại với nhau. Nhanh hơn nhiều!

2.2. Độ phức tạp

Trường hợp Độ phức tạp Ghi chú
Tốt nhấtO(n log n)Luôn như nhau
Trung bìnhO(n log n)Ổn định, không phụ thuộc dữ liệu
Xấu nhấtO(n log n)Khác Bubble Sort (O(n²))

2.3. Kỹ thuật 2 con trỏ (nhanh / chậm) để tìm điểm giữa

Để cắt danh sách làm 2, ta cần tìm điểm giữa. Cách dễ nhất là đếm số nút rồi đi nửa đường. Nhưng có cách khéo hơn: dùng 2 con trỏ:

  • nhanh — đi 2 bước mỗi lần
  • cham — đi 1 bước mỗi lần

Khi nhanh đi hết danh sách thì cham đang ở giữa!

ung_dung_giao_tiep/loi_thuat_toan/danh_sach_lien_ket.py
def _chia_doi(self, dau):
    nhanh = dau                            # Con trỏ nhanh đi 2 bước/lần
    cham  = dau                            # Con trỏ chậm đi 1 bước/lần
    while nhanh.sau is not None and nhanh.sau.sau is not None:
        nhanh = nhanh.sau.sau              # Nhanh đi 2 bước
        cham  = cham.sau                   # Chậm đi 1 bước

    giua = cham.sau                        # Điểm giữa = nút ngay sau `cham`
    cham.sau = None                        # Cắt đôi: chấm dứt nửa đầu
    return giua                            # Trả về đầu của nửa sau

2.4. Hàm đệ quy _merge_sort()

ung_dung_giao_tiep/loi_thuat_toan/danh_sach_lien_ket.py
def _merge_sort(self, dau, tieu_chi, tang_dan):
    # Điều kiện dừng đệ quy: 0 hoặc 1 nút thì đã sắp xếp rồi
    if dau is None or dau.sau is None:
        return dau

    # 1. CHIA — Cắt đôi danh sách
    giua = self._chia_doi(dau)

    # 2. GỌI ĐỆ QUY — Sắp xếp từng nửa
    trai = self._merge_sort(dau,  tieu_chi, tang_dan)
    phai = self._merge_sort(giua, tieu_chi, tang_dan)

    # 3. TRỘN — Gộp 2 nửa đã sắp xếp
    return self._tron(trai, phai, tieu_chi, tang_dan)
⚠️ Hiểu về đệ quy (recursion): Hàm _merge_sort gọi chính nó với input nhỏ hơn. Quá trình cứ vậy cho đến khi điều kiện dừng (dau is None hoặc chỉ còn 1 nút). Khi đó hàm trả về danh sách đã sắp xếp (1 nút thì coi như đã sắp xếp). Các lời gọi đệ quy sẽ dần dần trộn các phần nhỏ lại thành kết quả cuối.

2.5. Hàm trộn _tron()

ung_dung_giao_tiep/loi_thuat_toan/danh_sach_lien_ket.py
def _tron(self, trai, phai, tieu_chi, tang_dan):
    if trai is None: return phai
    if phai is None: return trai

    # Lấy giá trị để so sánh bằng hàm getattr (lấy thuộc tính động)
    gt_trai = getattr(trai.du_lieu, tieu_chi)
    gt_phai = getattr(phai.du_lieu, tieu_chi)

    # Tránh lỗi phân biệt chữ hoa/thường khi sắp xếp theo tên
    if isinstance(gt_trai, str):
        gt_trai  = gt_trai.lower()
        gt_phai  = gt_phai.lower()

    # So sánh 2 giá trị → quyết định nút nào đứng trước
    dieu_kien = gt_trai <= gt_phai if tang_dan else gt_trai >= gt_phai
    if dieu_kien:
        trai.sau = self._tron(trai.sau, phai, tieu_chi, tang_dan)
        trai.sau.truoc = trai
        trai.truoc = None
        return trai
    else:
        phai.sau = self._tron(trai, phai.sau, tieu_chi, tang_dan)
        phai.sau.truoc = phai
        phai.truoc = None
        return phai

Giải thích từng bước:

  • if trai is None: return phai — nếu nửa trái rỗng, trả luôn nửa phải (đã sắp xếp).
  • if phai is None: return trai — nếu nửa phải rỗng, trả luôn nửa trái.
  • getattr(trai.du_lieu, tieu_chi) — lấy thuộc tính động. Nếu tieu_chi="gia_ban" thì lấy trai.du_lieu.gia_ban. Nếu tieu_chi="ten_san_pham" thì lấy tên.
  • isinstance(gt_trai, str) — nếu giá trị là chuỗi, đổi về chữ thường để sắp xếp không phân biệt hoa/thường.
  • Phần dieu_kien — so sánh giá trị 2 nút, quyết định nút nào nhỏ hơn sẽ đứng trước.
  • Đệ quy: trai.sau = self._tron(trai.sau, phai, ...) — nút thắng sẽ trỏ sau tới kết quả trộn của phần còn lại.

3.1. Django Views LÀ GÌ?

Views (xử lý yêu cầu) là trái tim của ứng dụng Django. Khi người dùng mở trang web hoặc bấm nút, trình duyệt gửi 1 HTTP Request tới server. Django điều hướng request đó tới 1 hàm Python gọi là view. Hàm này xử lý rồi trả về HTTP Response (thường là JSON hoặc HTML).

💡 So sánh với PyQt6: Trong PyQt6, ta gắn sự kiện nút bấm bằng nut.clicked.connect(ham). Trong Django, ta gắn URL với hàm bằng path('duong-dan', ham). Khi user truy cập URL đó → Django tự gọi ham(request).

Giải thích từng dòng:

  • from django.http import JsonResponse — nhập lớp JsonResponse để trả về dữ liệu JSON (frontend sẽ nhận và render).
  • from django.shortcuts import render, redirect — render dùng để trả về trang HTML, redirect dùng để chuyển hướng sang URL khác.
  • @csrf_exempt — bỏ qua kiểm tra CSRF token (chống tấn công cross-site). Đồ án học nên bỏ qua để test dễ, thực tế phải giữ.
  • @require_http_methods(['POST']) — chỉ cho phép phương thức POST. Nếu user gửi GET → Django trả lỗi 405 Method Not Allowed.
  • import json — nhập thư viện json để parse body request thành dict Python.
  • 3 biến toàn cục Danh_sach_cua_hang, Bo_undoredo, Gio_hang_hien_tai — lưu trạng thái ứng dụng trên RAM. Khác PyQt6 (lưu trong self.xxx), Django không có self nên dùng biến toàn cục.

            )

        # Lưu thông tin vào session

        request.session['ho_ten']        = nguoi_dung.ho_ten

    except json.JSONDecodeError:
        return JsonResponse(
            {'trang_thai': 'loi', 'thong_bao': 'Dữ liệu gửi lên không hợp lệ.'},
            status=400

Giải thích từng dòng:

  • @csrf_exempt — decorator (bộ trang trí) bỏ qua CSRF. Decorator là hàm bọc hàm khác để thêm tính năng.
  • @require_http_methods(['POST']) — chỉ chấp nhận POST. GET bị từ chối.
  • json.loads(request.body) — parse body request (JSON string) thành dict Python.
  • du_lieu.get('ten_dang_nhap', '').strip() — lấy giá trị khóa ten_dang_nhap, nếu không có trả ''. .strip() cắt khoảng trắng đầu/cuối.

  • except json.JSONDecodeError — bắt lỗi nếu body không phải JSON hợp lệ.
  • except Exception as loi — bắt mọi lỗi khác để server không sập.

BÀI 5 — UNDO/REDO BẰNG 2 STACK (LIFO) HUY

5.1. Ý tưởng

Undo/Redo dùng 2 Stack:

  • Stack Undo — lưu các thao tác đã làm. Bấm Undo → lấy 1 phần tử ra.
  • Stack Redo — lưu các thao tác đã Undo. Bấm Redo → lấy 1 phần tử ra.
💡 Quy tắc vàng: Mỗi lần user làm thao tác mới → xóa sạch Stack Redo. Vì chuỗi thao tác đã đứt, không thể redo lại được nữa.

5.2. Cấu trúc 1 "thao tác" trong Stack

# Thao tác THÊM

# Thao tác SỬA

  • Nếu loai == 'xoa': ta đã XÓA. Để undo → phải THÊM lại. Lấy dữ liệu từ du_lieu_cu (đã lưu sẵn trước khi xóa).
  • Nếu loai == 'sua': ta đã SỬA. Để undo → khôi phục lại du_lieu_cu.
  • Bo_undoredo['redo'].day_vao(thao_tac) — đẩy thao tác sang Stack Redo. Nếu user bấm Redo, ta sẽ lấy từ đây.
